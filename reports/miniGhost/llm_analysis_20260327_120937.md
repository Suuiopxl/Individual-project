## 1. `mg_stencil_2d5pt`

### 1. 功能摘要
执行二维五点模板(stencil)计算，即对 GRID 的内层网格点应用邻居加权平均，结果写入 WORK，再复制回 GRID。

### 2. 计算特征
计算密集型，带有显著的内存访问需求（多次访问 GRID 周围点），且三层嵌套循环计算。

### 3. 数据依赖分析
不同网格点迭代间无依赖；两个三层循环依赖段间有数据依赖（WORK 必须先计算再使用）；循环内元素间无依赖。完全可并行。

### 4. GPU 加速可行性评分
9 / 10

### 5. 推荐 GPU 化方案
OpenACC（快速实现且易调试），CUDA（追求性能时）

### 6. 预期加速比
>5x - 20x （受限于内存带宽和核间同步，但二维 stencil 结构适合GPU）

### 7. 实现难度
中

### 8. 具体建议
- 将 GRID 和 WORK 分配到设备内存。
- 两个循环分别通过并行线程块执行，线程映射 (I,J,K)。
- 利用共享内存缓存临近点（如用 CUDA）提升带宽。
- 保持数据结构连续，避免间隔访问。
- OpenACC 示例伪代码：

```fortran
!$acc data copy(GRID(0:NX+1,0:NY+1,0:NZ+1)), create(WORK(0:NX+1,0:NY+1,0:NZ+1))
!$acc parallel loop collapse(3)
do k=1,NZ
  do j=1,NY
    do i=1,NX
      WORK(i,j,k) = (GRID(i-1,j,k)+GRID(i,j-1,k)+GRID(i,j,k)+GRID(i,j+1,k)+GRID(i+1,j,k))/5.0
    end do
  end do
end do

!$acc parallel loop collapse(3)
do k=1,NZ
  do j=1,NY
    do i=1,NX
      GRID(i,j,k) = WORK(i,j,k)
    end do
  end do
end do
!$acc end data
```

## 2. `mg_sum_grid`

### 1. 功能摘要
将 GRID 上所有有效网格点元素求和（含部分边界贡献）。

### 2. 计算特征
计算密集型，带有全局通信（MPI_Allreduce），求和属于归约操作。

### 3. 数据依赖分析
单步归约，迭代内无数据依赖，适合并行归约；但跨MPI rank间通信为性能瓶颈。

### 4. GPU 加速可行性评分
7 / 10

### 5. 推荐 GPU 化方案
OpenACC 归约或 CUDA Thrust归约，再行 MPI_Allreduce 通信。

### 6. 预期加速比
2x - 5x （计算部分加速显著，但 MPI_Allreduce 限制总速度）

### 7. 实现难度
中

### 8. 具体建议
- 使用 GPU 进行数据归约（核内归约+跨线程块归约），减少CPU负载。
- MPI通信保持CPU端实现或用 CUDA-aware MPI。
- 伪代码：

```fortran
!$acc data copyin(GRID), create(GSUM)
LSUM = 0.0
!$acc parallel loop collapse(3) reduction(+:LSUM)
do k=1,NZ
  do j=1,NY
    do i=1,NX
      LSUM = LSUM + GRID(i,j,k)
    end do
  end do
end do
LSUM = LSUM + FLUX_OUT(IVAR)

! MPI_Allreduce (LSUM -> GSUM)
```

- 若使用CUDA，使用Thrust或自定义reduce kernel实现。

## 3. `mg_flux_accumulate_2d5pt_3d7pt`

### 1. 功能摘要
沿进程边界对指定方向的 GRID 边界点执行加权累积，得到边界通量输出 FLUX_OUT。

### 2. 计算特征
内存密集型，涉及边界扫描累加；小范围数据（沿边界面）加法操作；无通信但比例较小。

### 3. 数据依赖分析
对 FLUX_OUT 变量的累加存在竞争，不同边界累积独立，可分开处理，需规避竞争。

### 4. GPU 加速可行性评分
4 / 10

### 5. 推荐 GPU 化方案
OpenACC（异步执行能快速实施），CUDA 用原子加速也可考虑；但整体收益有限。

### 6. 预期加速比
1.5x - 3x

### 7. 实现难度
中

### 8. 具体建议
- 将边界循环映射至GPU线程。
- 使用线程私有临时变量做局部累加，最后进行原子累加到全局 FLUX_OUT。
- 若多方向边界累加可分多步执行，避免原子冲突。
- 伪代码（OpenACC）：

```fortran
! 每个方向累加可单独并行
!$acc parallel loop reduction(+:FLUX_LOCAL)
do k=1,NZ
  do j=1,NY
    FLUX_LOCAL = FLUX_LOCAL + GRID(1,j,k)/DIVISOR
  end do
end do
! 使用原子将 FLUX_LOCAL 写回 FLUX_OUT(IVAR)
!$acc atomic
FLUX_OUT(IVAR) = FLUX_OUT(IVAR) + FLUX_LOCAL
```

# 总结

- **mg_stencil_2d5pt** 是首要GPU优化目标，采用 OpenACC 快速实现，进一步用 CUDA 优化共享内存利用，能显著提升性能。
- **mg_sum_grid** 适合GPU端归约后通过MPI_Allreduce完成全局归约，需覆盖CUDA-aware MPI。
- **mg_flux_accumulate_2d5pt_3d7pt** 优化收益有限，应在前两者优化完毕后再考虑。

针对 miniGhost 的问题规模(100³)，GPU加速效果明显，硬件资源合适时预期整体加速比可达5倍以上。
